# Подключение Telegram-бота для уведомлений о заказах

## Краткая схема работы

```
Покупатель → checkout → fetch API → Telegram Bot API → чат с менеджером
```

Клиент заполняет форму на `/checkout/` → скрипт отправляет сообщение через `api.telegram.org` → сообщение приходит в Telegram тому, кто обрабатывает заказы.

---

## Шаг 1. Создать бота через @BotFather

1. Открой Telegram → напиши `@BotFather`
2. Команда: `/newbot`
3. Введи название бота: `XO Diamonds Orders`
4. Введи username бота: `@xo_diamonds_bot` (или любой свободный)
5. BotFather выдаст **токен** (строка вида `123456789:ABCdefGHIjklMNOpqrsTUVwxyz`)

> 🔐 Сохрани токен — это пароль от бота.

---

## Шаг 2. Узнать Chat ID получателя

1. Напиши созданному боту любое сообщение (например «test»)
2. Открой в браузере:
   ```
   https://api.telegram.org/bot<ТВОЙ_ТОКЕН>/getUpdates
   ```
3. В ответе JSON найди: `"chat":{"id": 123456789}`
4. Запиши этот `chat_id` — это номер чата менеджера.

**Альтернативно** — напиши боту `@userinfobot`, он скажет твой chat_id.

---

## Шаг 3. Вписать токен и chat_id в код сайта

Файл: `src/pages/checkout.astro`

Найди строки **58–59**:

```js
const TELEGRAM_BOT_TOKEN = 'YOUR_BOT_TOKEN';
const TELEGRAM_CHAT_ID = 'YOUR_CHAT_ID';
```

Замени на реальные значения:

```js
const TELEGRAM_BOT_TOKEN = '123456789:ABCdefGHIjklMNOpqrsTUVwxyz';
const TELEGRAM_CHAT_ID = '123456789';
```

---

## Шаг 4. Пересобрать и задеплоить сайт

```bash
npm run build
# Затем деплой dist/ на хостинг
```

## Безопасность

- **Токен бота** нельзя публиковать в открытом репозитории.
- При использовании GitHub Actions токен можно передать через **Secrets**.
- Без реального токена checkout просто покажет сообщение об ошибке — сайт остаётся рабочим.

---

## Альтернативы

Вместо Telegram можно:

| Вариант | Как сделать |
|---|---|
| **Email** | Заменить fetch на отправку через form-to-email сервис (formspree.io, web3forms.com) |
| **WhatsApp** | Использовать WhatsApp Business API (сложнее, требует бизнес-аккаунт) |
| **Google Sheets** | Отправлять заказы в таблицу через Google Sheets API |

Формат данных заказа уже готов — достаточно заменить endpoint в `submitOrder()`.
